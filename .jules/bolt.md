## 2024-04-30 - Replace Regex with String Parsing in Hot Paths
**Learning:** Coordinate string keys (like `v_q_r_dir` and `e_q_r_dir`) are extremely hot paths in the game logic engine, frequently parsed when calculating things like longest road, adjacent buildings, and trading paths. Using standard Regex matching (`String.prototype.match`) is relatively expensive inside deep calculation loops. A custom string parsing loop character-by-character operates approximately 10x faster (400ms -> 34ms per 1M iterations) for this predictable coordinate format.
**Action:** When extracting components from coordinate/ID strings in hot loops (like graph traversal or distance calculations), use simple custom string parsing iterating over character codes instead of instantiating Regexes or using `.split()`.
